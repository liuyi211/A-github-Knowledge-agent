# OMKA 飞书接入修复计划：4.29.feishu_change

## 0. 当前目标

当前 OMKA 已完成上一阶段产品化计划。现在只修复和重构 **飞书接入方式**。

本次目标是：

```text
弃用 Feishu Webhook 自定义群机器人方案
改为 Feishu 开放平台自建应用机器人方案
使用 App ID + App Secret 获取 tenant_access_token
以应用机器人身份向飞书用户或群聊发送每日简报
预留后续“用户在飞书里发消息，与 OMKA 后端 Agent 对话”的扩展接口
```

本计划只处理飞书相关能力，不改动 GitHub 抓取、Digest 生成、知识库、推荐排序等主流程。

---

## 1. 本次修复边界

### 1.1 必须做

```text
P0 必做：
1. 移除当前飞书 Webhook 作为主路径的实现
2. 新增 Feishu App Bot 配置方式
3. 新增 App ID / App Secret / tenant_access_token 管理
4. 支持应用机器人向指定 chat_id 或 open_id 发送测试消息
5. 支持应用机器人推送最新 Digest
6. 后端新增飞书事件接收 endpoint，为后续飞书内对话预留入口

8. 新增 FeishuConversationGateway 抽象接口，为后续接入后端 Agent 预留
9. UI Settings 页面改为飞书应用机器人配置，不再展示 Webhook 配置
10. 所有飞书常量、密钥、开关、回调地址都放在 .env 中
11. 记录飞书发送日志、事件日志、token 获取日志
12. 对 App Secret、Verification Token、Encrypt Key 做脱敏显示和日志保护
```

本阶段只做：


## 2. 飞书接入方案调整

### 2.1 原方案废弃

原方案：

```text
飞书自定义群机器人 Webhook
FEISHU_WEBHOOK_URL
FEISHU_WEBHOOK_SECRET
直接 POST Webhook URL 推送消息
```

本次改为：

```text
飞书开放平台自建应用机器人
FEISHU_APP_ID
FEISHU_APP_SECRET
tenant_access_token
im/v1/messages 发送消息
事件订阅 endpoint 接收用户消息
```

---

### 2.2 新方案架构

```text
OMKA Digest
   ↓
FeishuNotificationService
   ↓
FeishuAppBotClient
   ↓
FeishuAuthService 获取/缓存 tenant_access_token
   ↓
Feishu OpenAPI Send Message
   ↓
飞书群聊 / 用户私聊
```

后续飞书内对话预留：

```text
飞书用户消息
   ↓
POST /api/integrations/feishu/events
   ↓
FeishuEventHandler
   ↓
FeishuCommandRouter
   ↓
FeishuConversationGateway
   ↓
OMKA Backend Agent
   ↓
FeishuAppBotClient.reply_message()
```

本阶段可以只实现 `FeishuConversationGateway` 的接口和空实现，不正式接入 Agent。

## 5. 核心类规划

### 5.1 FeishuConfig

负责从 env 读取飞书配置。

```python
class FeishuConfig(BaseModel):
    enabled: bool
    app_id: str | None
    app_secret: str | None
    verification_token: str | None
    encrypt_key: str | None
    api_base_url: str
    request_timeout_seconds: int
    max_retries: int

    default_receive_id_type: str
    default_chat_id: str | None

    command_prefix: str
    require_mention: bool
    group_allowlist: list[str]
    user_allowlist: list[str]

    push_digest_enabled: bool
    push_digest_top_n: int

    public_callback_url: str | None

    agent_conversation_enabled: bool
    agent_session_ttl_minutes: int
    agent_max_message_chars: int
```

要求：

```text
1. 统一从 env 读取
2. 不允许散落 os.getenv
3. Settings API 使用这个配置对象返回脱敏状态
4. app_secret、verification_token、encrypt_key 只显示 configured: true/false
```

---

### 5.2 FeishuAuthService

负责获取和缓存 `tenant_access_token`。

接口：

```python
class FeishuAuthService:
    async def get_tenant_access_token(self) -> str:
        pass

    async def refresh_tenant_access_token(self) -> str:
        pass
```

要求：

```text
1. token 缓存到数据库或内存均可，优先数据库
2. 记录 expires_at
3. token 过期前提前刷新
4. token 获取失败要返回用户可理解错误
5. 不在日志中打印 app_secret 或 token
```

---

```python
        pass

```

### 5.4 FeishuNotificationService

负责把 OMKA Digest 转成飞书消息。

接口：

```python
class FeishuNotificationService:

    async def send_latest_digest(self, receive_id: str | None = None) -> SendResult:
        pass

```

## 10. 权限和安全要求

### 10.1 最小权限

飞书应用只申请本阶段必须权限：

```text
1. 启用机器人能力
2. 发送消息能力
3. 接收消息事件能力，如果要接收 /omka 命令
```

不要申请：

```text
1. 云文档读取权限
2. 日历权限
3. 通讯录批量读取权限
4. 用户敏感信息权限
5. 审批权限
6. 管理员权限
```

---

### 10.2 Secret 保护

```text
1. App Secret 不得进入前端明文
2. tenant_access_token 不得进入前端
3. 日志中不得打印 App Secret
4. 日志中不得打印 tenant_access_token
5. 错误信息中不得输出完整请求头
6. chat_id、open_id 在日志中脱敏
```

---

### 10.3 事件安全

```text
1. 校验 verification token
2. 支持 challenge 验证
3. 预留 encrypt_key 解密逻辑
4. 只处理 allowlist 中的群或用户
5. requireMention=true 时，群聊中未 @ 机器人则忽略
6. 防止重复 event_id 重复处理
```

---
