# 4.29 Feishu Change 1：飞书单聊 + 简单 Agent 自由聊天升级计划

> 本计划只更改飞书相关能力。  
> 目标是在已有 OMKA 项目基础上，将飞书从“群聊/推送渠道”升级为“单聊机器人 + 简单 Agent 对话入口”。  
> 这次必须体现“智能知识助手”能力：用户可以在飞书单聊中直接发自然语言问题，后端用一个简单 Agent 结合已有 Digest / Knowledge / GitHub 候选内容进行回答。

---

## 1. 本次升级目标

当前飞书接入不再走：

```text
飞书群聊推送
飞书自定义群机器人 Webhook
只发每日简报、不接收用户消息
```

改为：

```text
飞书开放平台应用机器人
用户与机器人单独聊天
机器人接收用户消息
OMKA 后端处理消息
简单 Agent 生成回答
机器人回复到同一个飞书单聊窗口
```

本次升级完成后的效果：

```text
用户：你好
机器人：你好，我是 OMKA，你的个人知识助手。你可以问我今天的技术情报、已收藏知识、GitHub 项目推荐，也可以使用 /omka help 查看命令。

用户：今天有什么 AI Agent 方向值得关注？
机器人：根据今日 GitHub 简报和你的兴趣，今天最值得关注的是 xxx、xxx、xxx。原因是...

用户：帮我总结一下最近收藏的 LangGraph 相关内容
机器人：你最近收藏的 LangGraph 相关知识主要包括...
```

## 2. 本次只改飞书，不改其他主流程

本次升级范围只包含：

```text
飞书应用机器人
飞书单聊消息接收
飞书单聊消息回复
飞书单聊会话绑定

飞书事件日志
飞书消息日志
简单 Agent 对话模块
Agent 使用已有 Digest / Candidate / Knowledge 数据回答
```

严禁改动：

```text
GitHub 抓取流程
GitHub 抓取范围
Digest 生成主流程
Knowledge 入库主流程
推荐排序主逻辑
前端整体视觉主题
其他信息源接入
数据库整体重构
多用户账号系统

```

允许做的连接：

```text
Agent 可以只读使用已有数据：
1. 最新 Digest
2. Candidate 推荐内容
3. Knowledge 已收藏内容
4. Job 状态
5. 用户兴趣 / 项目配置
```

## 5. Agent 能力定位

本阶段不要做复杂 Agent 框架。

本阶段 Agent 定义为：

```text
一个轻量的知识问答 Agent
能够理解用户自然语言问题
能够读取 OMKA 当前已有知识数据
能够生成有上下文的回答
能够在回答中说明来源
```

命名：

```text
SimpleKnowledgeAgent
```

它不是：

```text
多 Agent 系统
自主执行工具的 Agent
浏览器 Agent
代码执行 Agent
自动写数据库的 Agent
```

## 6. SimpleKnowledgeAgent 设计

### 6.1 模块路径

新增：

```text
app/agents/
  base.py
  simple_knowledge_agent.py
  context_builder.py
  prompts.py
  schemas.py
```

### 6.2 Agent 接口

```python
class AgentResponse(BaseModel):
    answer: str
    used_context: list[dict] = []
    suggested_actions: list[str] = []
    confidence: float | None = None


class AgentContext(BaseModel):
    user_message: str
    channel: str
    conversation_id: str
    user_external_id: str
    recent_messages: list[dict] = []
    latest_digest_items: list[dict] = []
    knowledge_items: list[dict] = []
    candidate_items: list[dict] = []
    user_profile: dict = {}


class BaseAgent:
    async def answer(self, context: AgentContext) -> AgentResponse:
        raise NotImplementedError
```

### 6.3 SimpleKnowledgeAgent 输入

```text
1. 用户当前消息
2. 最近 N 条对话消息
3. 最新 Digest Top N
4. Knowledge 搜索结果 Top N
5. Candidate 搜索结果 Top N
6. 用户兴趣配置
7. 用户当前项目配置
```

### 6.4 SimpleKnowledgeAgent 输出

```text
1. answer：给用户的自然语言回答
2. used_context：使用了哪些知识上下文
3. suggested_actions：建议用户下一步做什么
4. confidence：可选置信度
```

### 6.5 回答要求

Agent 回答必须：

```text
1. 简洁
2. 直接回答用户问题
3. 优先基于 OMKA 已有数据
4. 不知道时明确说不知道
5. 不编造不存在的 GitHub 项目或知识
6. 尽量说明信息来自今日简报、收藏知识或候选池
7. 不输出过长内容
```

## 7. Agent 上下文构建

新增：

```text
app/agents/context_builder.py
```

职责：

```text
1. 根据用户问题搜索 Knowledge
2. 根据用户问题匹配最新 Digest
3. 根据用户问题匹配 Candidate
4. 读取用户兴趣和项目配置
5. 读取最近对话消息
6. 裁剪上下文长度
```

### 7.1 MVP 检索方式

本阶段不引入向量数据库。

使用简单规则：

```text
1. 关键词匹配
2. 标题匹配
3. 标签匹配
4. 项目名匹配
5. 最近时间优先
```

不要做：

```text
1. 向量库
2. embedding
3. reranker
4. 知识图谱
```

### 7.2 上下文数量限制

通过 `.env` 控制：

```env
OMKA_AGENT_MAX_RECENT_MESSAGES=6
OMKA_AGENT_MAX_DIGEST_ITEMS=5
OMKA_AGENT_MAX_KNOWLEDGE_ITEMS=5
OMKA_AGENT_MAX_CANDIDATE_ITEMS=5
OMKA_AGENT_MAX_CONTEXT_CHARS=12000
```

## 8. Agent Prompt 设计

新增：

```text
app/agents/prompts.py
```

建议系统 Prompt：

```text
你是 OMKA，一个个人智能知识助手。

你需要根据用户的兴趣、当前项目、最新技术简报、已收藏知识和候选内容回答问题。

规则：
1. 优先使用提供的上下文回答。
2. 如果上下文不足，请明确说明“不确定”或“当前知识库中没有足够信息”。
3. 不要编造 GitHub 项目、数据、Release 或用户收藏。
4. 回答要简洁、实用，适合飞书聊天阅读。
5. 如果用户问“今天有什么值得关注”，优先参考最新 Digest。
6. 如果用户问“我收藏过什么”，优先参考 Knowledge。
7. 如果用户问“这个方向有哪些候选”，优先参考 Candidate。
8. 可以给出建议动作，但不要替用户做投资建议、法律建议、医疗建议。
9. 输出中文。
```

用户 Prompt 拼接：

```text
用户问题：
{user_message}

用户兴趣：
{interests}

当前项目：
{projects}

最近对话：
{recent_messages}

最新简报：
{latest_digest_items}

已收藏知识：
{knowledge_items}

候选内容：
{candidate_items}
```

## 15. 消息处理流程

### 15.1 命令消息流程

```text
飞书用户发送 /omka latest
→ 飞书事件
→ FeishuEventReceiver
→ 判断单聊
→ 保存 feishu_event_logs
→ CommandRouter
→ 执行 latest
→ FeishuClient 回复
→ 保存 feishu_message_runs
```

### 15.2 自由聊天流程

```text
飞书用户发送普通文本
→ 飞书事件
→ FeishuEventReceiver
→ 判断单聊
→ 保存 feishu_event_logs
→ 保存 conversation_messages(role=user)
→ CommandRouter 判断不是命令
→ ConversationService
→ SimpleKnowledgeAgent
→ ContextBuilder 构建上下文
→ 调用 LLM
→ 保存 agent_runs
→ 保存 conversation_messages(role=assistant)
→ FeishuClient 回复
→ 保存 feishu_message_runs
```

### 15.3 Agent 失败兜底

如果 Agent 调用失败，回复：

```text
我刚才处理你的问题时失败了。你可以稍后再试，或者使用 /omka latest 查看最新简报。
```

如果上下文不足，回复：

```text
当前知识库里没有足够信息支持回答这个问题。你可以先收藏相关内容，或让我查看最新简报。
```
