## 一、快速开始

### 1. 启动服务

```bash
# 进入项目目录
cd E:\GitHub\Repositories\oh-my-knowledge-assistant

# 激活 conda 环境
conda activate omk

# 启动服务
python -m uvicorn omka.app.main:app --reload --host 0.0.0.0 --port 8000
```

看到以下日志即表示启动成功：

```
INFO:     Uvicorn running on http://0.0.0.0:8000
INFO:     Application startup complete.
OMKA 启动完成 | API=http://0.0.0.0:8000
```

### 2. 访问 API 文档（Swagger UI）

```
http://localhost:8000/docs
```

**注意**：不要用 `0.0.0.0:8000` 访问，浏览器无法识别 `0.0.0.0` 这个地址。请使用：

- `http://localhost:8000/docs` — API 文档 + 在线测试
- `http://localhost:8000/health` — 健康检查
- `http://localhost:8000/redoc` — 备用 API 文档（ReDoc 风格）

## 二、为什么浏览器访问不了

### 问题现象

你在浏览器地址栏输入 `0.0.0.0:8000`，显示"无法访问此网站"。

### 原因

`0.0.0.0` 是一个**特殊地址**，表示"监听所有网络接口"，但它本身不是一个有效的目标地址。浏览器不知道要连接到哪个具体的 IP。

### 正确访问方式

| 场景 | 访问地址 |
|------|---------|
| 本机浏览器访问 | `http://localhost:8000` 或 `http://127.0.0.1:8000` |
| 同一局域网其他设备 | `http://<你的电脑IP>:8000`（如 `http://192.168.1.5:8000`） |
| API 文档 | `http://localhost:8000/docs` |

> 查看本机 IP：`ipconfig`（Windows）或 `ifconfig`（Linux/macOS）

## 三、OMKA 是什么

OMKA 是一个**后端 API 服务**，没有可视化前端界面。它的核心功能是：

1. **自动抓取** GitHub 仓库、Release、搜索结果
2. **智能排序** 根据你的兴趣关键词打分排名
3. **生成简报** 每日生成 Markdown 格式的知识简报
4. **知识管理** 将感兴趣的内容存入个人知识库

### 数据流转

```
配置文件 (sources.yaml)
    |
    v
数据源配置 → 抓取(fetcher) → 原始数据(RawItem)
    |
    v
清洗归一化(cleaner) → 标准化数据(NormalizedItem)
    |
    v
去重评分(deduper + ranker) → 候选条目(CandidateItem)
    |
    v
生成简报(digest_builder) → Markdown 文件
    |
    v
用户确认 → 知识库(KnowledgeItem)
```

## 四、核心概念

### 1. 数据源（Source）

配置要抓取的 GitHub 内容来源：

- **repo 模式**：监控指定仓库（获取仓库信息和最新 Release）
- **search 模式**：按关键词搜索 GitHub 仓库

### 2. 候选池（Candidate）

系统抓取并排序后的推荐内容，状态有：

- `pending` — 待处理（默认）
- `confirmed` — 已确认（入库到知识库）
- `ignored` — 已忽略

### 3. 知识库（Knowledge）

你确认感兴趣的内容会永久保存到知识库，同时生成 Markdown 文件到 `data/knowledge/` 目录。

### 4. 每日简报（Digest）

系统每天自动运行的完整流程：抓取 → 清洗 → 去重 → 排序 → 生成 Markdown 简报。

## 五、API 接口使用指南

所有 API 都以 `http://localhost:8000` 为基地址。

### 1. 系统接口

#### 健康检查

```bash
curl http://localhost:8000/health
```

响应：
```json
{
  "status": "ok",
  "app": "OMKA",
  "version": "0.1.0",
  "env": "development"
}
```

---

### 2. 数据源管理（/sources）

#### 列出所有数据源

```bash
curl http://localhost:8000/sources
```

#### 创建数据源 — repo 模式

```bash
curl -X POST http://localhost:8000/sources \
  -H "Content-Type: application/json" \
  -d '{
    "id": "facebook-react",
    "source_type": "github",
    "name": "React 官方仓库",
    "mode": "repo",
    "repo_full_name": "facebook/react",
    "weight": 1.5
  }'
```

#### 创建数据源 — search 模式

```bash
curl -X POST http://localhost:8000/sources \
  -H "Content-Type: application/json" \
  -d '{
    "id": "search-fastapi",
    "source_type": "github",
    "name": "FastAPI 相关仓库",
    "mode": "search",
    "query": "fastapi stars:>1000",
    "limit": 10,
    "weight": 1.0
  }'
```

#### 手动运行单个数据源

```bash
curl -X POST http://localhost:8000/sources/facebook-react/run
```

#### 删除数据源

```bash
curl -X DELETE http://localhost:8000/sources/search-fastapi
```

---

### 3. 候选池管理（/candidates）

#### 列出候选条目

```bash
# 全部
curl http://localhost:8000/candidates

# 只看待处理
curl "http://localhost:8000/candidates?status=pending"

# 只看已确认
curl "http://localhost:8000/candidates?status=confirmed"
```

#### 确认入库（添加到知识库）

```bash
curl -X POST http://localhost:8000/candidates/<candidate_id>/confirm
```

#### 忽略条目

```bash
curl -X POST http://localhost:8000/candidates/<candidate_id>/ignore
```

#### 提交反馈

```bash
curl -X POST http://localhost:8000/candidates/<candidate_id>/feedback \
  -H "Content-Type: application/json" \
  -d '{
    "feedback_type": "not_interested",
    "notes": "已了解，不再推荐"
  }'
```

`feedback_type` 可选值：`confirm` / `ignore` / `not_interested` / `read_later`

---

### 4. 每日简报（/digests）

#### 运行完整每日任务

```bash
curl -X POST http://localhost:8000/digests/run-today
```

这会执行完整流程：抓取 → 清洗 → 去重 → 排序 → 生成简报。

#### 运行排序（重新计算分数）

```bash
curl -X POST http://localhost:8000/digests/run-ranking
```

#### 获取排名结果

```bash
curl "http://localhost:8000/digests/ranked?limit=20"
```

---

### 5. 知识库管理（/knowledge）

#### 列出知识库

```bash
curl http://localhost:8000/knowledge
```

#### 查看单条详情

```bash
curl http://localhost:8000/knowledge/<item_id>
```

#### 删除知识条目

```bash
curl -X DELETE http://localhost:8000/knowledge/<item_id>
```

## 六、日常使用工作流

### 场景 1：首次配置

1. 编辑 `data/profiles/sources.yaml` 添加关注的数据源（或调用 API 创建）
2. 编辑 `data/profiles/interests.yaml` 设置你的兴趣关键词
3. 启动服务
4. 调用 `/digests/run-today` 手动运行一次完整流程
5. 查看 `/candidates` 浏览推荐内容
6. 对感兴趣的内容调用 `/candidates/{id}/confirm` 确认入库

### 场景 2：每日使用

系统每天 09:00（上海时区）自动运行。你可以：

```bash
# 查看今日推荐
curl "http://localhost:8000/digests/ranked?limit=10"

# 确认感兴趣的第 1 条
curl -X POST http://localhost:8000/candidates/<id>/confirm

# 忽略不感兴趣的
curl -X POST http://localhost:8000/candidates/<id>/ignore

# 查看已收集的知识
curl http://localhost:8000/knowledge
```

### 场景 3：手动触发抓取

```bash
# 触发单个数据源
curl -X POST http://localhost:8000/sources/facebook-react/run

# 或触发完整流程
curl -X POST http://localhost:8000/digests/run-today
```

## 七、配置文件说明

### 1. 数据源配置：`data/profiles/sources.yaml`

```yaml
sources:
  - id: github-trending-python
    source_type: github
    name: "Python 热门仓库"
    enabled: true
    mode: search
    query: "language:python stars:>1000 sort:updated"
    limit: 10
    weight: 1.0

  - id: facebook-react
    source_type: github
    name: "React"
    enabled: true
    mode: repo
    repo_full_name: "facebook/react"
    weight: 1.5
```

### 2. 兴趣配置：`data/profiles/interests.yaml`

```yaml
interests:
  - name: "AI/ML"
    keywords: ["machine learning", "deep learning", "AI", "LLM", "神经网络"]
    weight: 2.0

  - name: "Python"
    keywords: ["python", "fastapi", "django", "flask"]
    weight: 1.5

projects:
  - name: "个人博客"
    keywords: ["blog", "markdown", "static site"]
    weight: 1.0
```

> 修改配置文件后**无需重启服务**，系统会读取数据库中已加载的配置。如要新增数据源，调用 API 或修改 YAML 后重启。

## 八、环境变量配置

复制 `.env.example` 为 `.env`，按需修改：

```bash
# GitHub Token（必须）
GITHUB_TOKEN=ghp_xxxxxxxxxxxxxxxxxxxx

# LLM 配置（用于生成摘要）
LLM_PROVIDER=openai
LLM_API_KEY=sk-xxxxxxxxxxxxxxxxxxxx
LLM_MODEL=gpt-4o-mini

# 本地 Ollama（可选，免费）
# LLM_PROVIDER=ollama
# OLLAMA_BASE_URL=http://localhost:11434
# OLLAMA_MODEL=qwen2.5:7b

# 每日任务时间（默认 09:00）
SCHEDULER_DAILY_CRON=0 9 * * *
SCHEDULER_TIMEZONE=Asia/Shanghai

# 排序权重（可调）
SCORE_WEIGHT_INTEREST=0.40
SCORE_WEIGHT_PROJECT=0.30
SCORE_WEIGHT_FRESHNESS=0.15
SCORE_WEIGHT_POPULARITY=0.15

# 简报数量
DIGEST_TOP_N=10
```

## 九、常见问题排查

### Q1：浏览器访问 `0.0.0.0:8000` 报错

**原因**：`0.0.0.0` 不是有效的浏览器目标地址。  
**解决**：使用 `http://localhost:8000` 或 `http://127.0.0.1:8000`。

### Q2：Swagger UI 在哪里

访问：`http://localhost:8000/docs`

### Q3：没有前端界面，怎么用

OMKA 是纯后端 API 服务。推荐用法：
- 浏览器打开 Swagger UI (`/docs`) 进行可视化操作
- 用 curl / Postman / HTTPie 调用 API
- 自己开发一个前端页面调用这些 API

### Q4：GitHub API 限流

未提供 Token 时，GitHub 限制 60 请求/小时。  
**解决**：在 `.env` 中设置 `GITHUB_TOKEN`。

### Q5：如何修改每日任务时间

修改 `.env`：
```bash
SCHEDULER_DAILY_CRON=0 9 * * *   # 每天 9:00
SCHEDULER_DAILY_CRON=0 20 * * *  # 每天 20:00
```

### Q6：LLM 摘要失败

检查 `.env` 中的 LLM 配置。如果没有 API Key，可切换到 Ollama 本地模型（免费）：

```bash
# 安装 Ollama
# 下载模型
ollama pull qwen2.5:7b

# 修改 .env
LLM_PROVIDER=ollama
OLLAMA_BASE_URL=http://localhost:11434
OLLAMA_MODEL=qwen2.5:7b
```

### Q7：数据库文件在哪里

```
data/db/app.sqlite
```

可用 SQLite 浏览器或命令行查看：
```bash
sqlite3 data/db/app.sqlite
.tables
select * from source_configs;
```

### Q8：简报文件输出到哪里

```
data/digests/           # 每日简报 Markdown
data/knowledge/github/  # 确认入库的知识条目 Markdown
```
